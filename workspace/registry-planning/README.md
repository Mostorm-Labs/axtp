# Registry Planning Workspace

This workspace contains maintainer planning material for registry taxonomy, candidate ids, and legacy classification.

These files are not runtime contracts and are not included in the default spec release artifact. Formal implementation facts must be adopted into `contract/registry/**`, generated into `contract/protocol/axtp.protocol.yaml` and `contract/generated/**`, and covered by conformance when required.

| Path | Use |
|---|---|
| `domain-feature-classification.md` | Long-form domain/feature classification guidance and legacy examples. |
| `candidates/*.md` | Historical and candidate method/event/error/profile/capability planning tables. |

For formal rules, use `specs/30-registry.md`, `specs/40-codec.md`, and `specs/50-tooling.md`.
