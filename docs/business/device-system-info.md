# <10-device-system-info>

## 背景

设备信息和系统状态监测是设备管理的基础接口，连接到设备的第一件事就是获取设备当前的设备状态以及基础的设备信息，设备信息包含设备id、sn码、产品信息、硬件信息、软件信息、上位机比如axtp自身的信息等，而系统system则管理了设备的运行状态，比如cpu、内存、在线、运行态等运行时状态信息，包括关机、重启等系统状态的控制

## 用户目标

用户想在连接到设备后，能够获取到设备的基础信息以及运行时状态参数，同时监控到设备的信息、状态所发生的变化

## 范围

设备信息检测、设备系统状态监测

## 非目标

除device 和 system之外的内容

## 场景

- 设备连接完成后，获取设备信息
- 设备连接完成后，获取主设备以及级连设备的信息
- 控制设备的启停
- 设备关机和重启控制
- 系统计划重启、计划关机任务的下发
- 重置设备状态，比如一些mcu设备，需要由上位机来指定重置，否则会处在一些异常的状态中，或者可以让设备从异常中恢复过来
- 重置设备到默认配置状态，属于软重置的一种：恢复到当前已安装版本的默认配置状态，不改变 Launcher 等软件自身版本。
- 重置设备为出厂设置状态，属于软重置的一种：恢复到设备出厂基线，可包含将 Launcher 等软件自身版本恢复到最初出厂版本。
